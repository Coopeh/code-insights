## Context

Tier 1 (PR #95, merged) surfaced rich metadata in type-specific insight cards and enhanced session rows. Tier 2 adds structural/functional improvements: sidebar layout, view modes, URL-persisted filters, conversation search, recurring patterns, and a progress ring.

**All items are frontend-only** — no API or schema changes needed. Estimated: 5 new files, ~5 modified files, ~595 new lines.

## Item 1: URL-Persisted Filters (FOUNDATION)

**Why first:** Items 3, 4, 6 depend on URL state for navigation links and view mode persistence.

**Approach:** Create `useFilterParams<T>` hook wrapping React Router's `useSearchParams`. Only non-default values written to URL (clean URLs). Search input uses `replace: true` to avoid history pollution; dropdowns use `push`.

**Files:**
| File | Action |
|------|--------|
| `dashboard/src/hooks/useFilterParams.ts` | **NEW** (~50 lines) — generic hook |
| `dashboard/src/pages/SessionsPage.tsx` | Replace 5 `useState` calls with `useFilterParams` |
| `dashboard/src/pages/InsightsPage.tsx` | Replace 4 `useState` calls with `useFilterParams` |

**Persisted filters:**
- SessionsPage: `q`, `project`, `source`, `character`, `status`
- InsightsPage: `q`, `project`, `type`, `view` (new view mode param)

## Item 4: Insights Page View Modes

**Approach:** Segmented control using existing shadcn `Tabs` component. Four modes with client-side grouping:

| Mode | Grouping | Section Headers |
|------|----------|-----------------|
| Timeline (default) | By date (Today/Yesterday/This Week/Earlier) | Date labels with count |
| By Type | By `insight.type` | "Decisions (8)", "Learnings (12)", etc. |
| By Project | By `project_name` | Project name + insight count |
| By Session | By `session_id` | Project name + timestamp |

Extract `getDateGroup` from SessionsPage to shared utility (reuse in both pages).

**Files:**
| File | Action |
|------|--------|
| `dashboard/src/pages/InsightsPage.tsx` | Add Tabs-based mode switcher + grouping logic (~150 lines added) |
| `dashboard/src/lib/utils.ts` | Extract `getDateGroup` helper from SessionsPage |
| `dashboard/src/pages/SessionsPage.tsx` | Import shared `getDateGroup` instead of inline |

## Item 5: Conversation Search

**Approach:** Client-side search. When user types in search bar, load all remaining messages (loop `fetchNextPage` until done), then filter by `message.content.includes(term)`.

**UX:** Search bar sticky at top of Conversation tab. Shows "N matches" with prev/next navigation buttons. Matching messages get a highlighted border/background (NOT inline text highlighting — that's Tier 3). Scrolls to current match via `scrollIntoView`.

**Files:**
| File | Action |
|------|--------|
| `dashboard/src/components/chat/conversation/ConversationSearch.tsx` | **NEW** (~80 lines) |
| `dashboard/src/components/chat/conversation/ChatConversation.tsx` | Accept `highlightMessageId` prop |
| `dashboard/src/pages/SessionDetailPage.tsx` | Add search state, wire to ConversationSearch |

---

## Item 6: Recurring Patterns Section

**Approach:** Client-side grouping from existing `linked_insight_ids` field. Group insights that share linked IDs. Show a "Recurring Patterns" section at top of InsightsPage (above view mode content) when groups exist.

Each pattern card shows: theme (title), occurrence count, project names, last seen date, "View all" button that filters the list via URL param (`?pattern=<groupKey>`).
